

## 1. LEFT SIDE PANEL STRUCTURE

### Text Elements Library Organization

The left panel organizes text elements into a hierarchical, searchable structure with visual previews and intelligent categorization.

#### **Panel Header**
- **Search Bar**: Full-text search across all text element types, names, and tags
- **Filter Dropdown**: Categories (All, Basic Text, Headings, Lists, Artistic, Dynamic)
- **Sort Options**: Recently Used, Alphabetical, Most Popular
- **View Toggle**: Grid view (larger previews) vs. List view (compact)

#### **Quick Access Section (Always Visible)**
- **Recently Used**: Last 5 text element types used, auto-updating
- **Favorites**: User-starred text elements for quick access
- **Current Document Styles**: Text styles already in use on the current canvas

## 2. RIGHT SIDE PANEL PROPERTIES ORGANIZATION

When any text element is selected, the right panel displays all editable properties organized into collapsible sections. Properties progress from basic (most frequently used) to advanced.

### **Section 1: Content (Always Expanded by Default)**

#### **Text Content Field**
- **Control Type**: Multi-line textarea (for multi-line text) or single-line input (for headings)
- **Functionality**: Direct text editing alternative to canvas editing
- **Features**:
  - Character count display (live)
  - Word count display
  - "Edit on Canvas" button to jump to inline editing
  - Spell check toggle
  - Find & replace button (opens modal)
- **Validation**: No character limit unless specified by element type
- **Dependencies**: Changes reflect immediately on canvas

#### **Text Style Preset Dropdown**
- **Control Type**: Dropdown menu with visual previews
- **Options**: All saved text styles + "Create New Style" option
- **Preview**: Each option shows font, size, and color preview
- **Functionality**: Apply predefined style bundle
- **Special**: "Update Style" button appears if current selection differs from saved style

---

### **Section 2: Typography (Collapsed by Default)**

#### **Font Family**
- **Control Type**: Searchable dropdown with font previews
- **Organization**:
  - Recently Used (top section)
  - System Fonts
  - Google Fonts (with search)
  - Uploaded Custom Fonts
  - Font categories: Serif, Sans-serif, Monospace, Handwriting, Display
- **Preview**: Each font name rendered in its own typeface
- **Default**: Arial/Helvetica (system default)
- **Dependencies**: Affects available weights and styles
- **Features**:
  - Favorite fonts (star icon)
  - Filter by language support
  - Preview text customization
- **Keyboard**: Up/down arrows to navigate, Enter to select

#### **Font Size**
- **Control Type**: Numeric input with slider (dual control)
- **Input Field**: Direct number entry
- **Slider**: 6px to 400px range
- **Default**: Varies by element type (16px for body, 48px for H1)
- **Unit**: px (pixels) - dropdown allows pt, em, rem
- **Increment**: 1px with arrows, 0.1px with shift+arrow
- **Validation**: 1px minimum, 1000px maximum
- **Quick Sizes**: Buttons for common sizes (12, 14, 16, 18, 24, 36, 48, 72)
- **Keyboard Shortcut**: Cmd/Ctrl + Shift + > to increase, < to decrease
- **Dependencies**: Affects text box auto-sizing
- **Visual Feedback**: Live preview on canvas while dragging slider

#### **Font Weight**
- **Control Type**: Dropdown or segmented slider (depending on font support)
- **Options**: Thin (100), Extra Light (200), Light (300), Regular (400), Medium (500), Semi-Bold (600), Bold (700), Extra Bold (800), Black (900)
- **Display**: Only shows weights available for selected font
- **Default**: 400 (Regular) for body, 700 (Bold) for headings
- **Keyboard Shortcut**: Cmd/Ctrl + B toggles bold (switches between 400 and 700)
- **Fallback**: If selected weight unavailable, uses nearest available

#### **Font Style**
- **Control Type**: Toggle buttons in horizontal group
- **Options**:
  - **Italic** (I icon) - Toggle button
  - **Underline** (U icon) - Toggle button with dropdown for style (single, double, dotted, dashed)
  - **Strikethrough** (S icon) - Toggle button
  - **Superscript** (x² icon) - Radio with subscript (mutually exclusive)
  - **Subscript** (x₂ icon) - Radio with superscript
  - **Small Caps** (TT icon) - Toggle button
- **Default**: All off
- **Keyboard Shortcuts**:
  - Cmd/Ctrl + I: Italic
  - Cmd/Ctrl + U: Underline
  - Cmd/Ctrl + Shift + X: Strikethrough
- **Dependencies**: Some effects incompatible with certain fonts

#### **Text Transform**
- **Control Type**: Dropdown
- **Options**:
  - None (original case)
  - UPPERCASE
  - lowercase
  - Capitalize Each Word
  - sWaP cAsE
- **Default**: None
- **Function**: Transforms display without changing actual text content
- **Reversible**: Original text preserved

#### **Letter Spacing (Tracking)**
- **Control Type**: Numeric input + slider
- **Range**: -200px to 200px
- **Default**: 0px
- **Unit**: px or em (toggleable)
- **Precision**: 0.1px increments
- **Visual**: Slider with detents at 0 (default)
- **Common Presets**: Tight (-0.5px), Normal (0), Loose (0.5px), Very Loose (2px)
- **Use Case**: Tightening for headlines, loosening for all-caps text

#### **Line Height (Leading)**
- **Control Type**: Numeric input + slider
- **Range**: 0.5 to 5.0 (multiplier) or absolute pixel values
- **Default**: 1.5 for body, 1.2 for headings
- **Unit**: Relative (1.5) or absolute (24px) - toggleable
- **Display**: Shows both multiplier and resulting pixel height
- **Visual**: Live preview of line spacing on canvas
- **Recommendation Indicator**: Green highlight when in optimal range (1.4-1.6 for body text)

#### **Paragraph Spacing**
- **Control Type**: Two numeric inputs (before and after)
- **Before Paragraph**: 0-100px, default 0px
- **After Paragraph**: 0-100px, default varies (16px for body)
- **Unit**: px
- **Quick Presets**: None, Tight (8px), Normal (16px), Loose (24px)
- **Visual**: Shows spacing guides on canvas when selected

---

### **Section 3: Alignment and Position**

#### **Horizontal Alignment**
- **Control Type**: Four icon buttons in horizontal group
- **Options**:
  - Align Left (⫰ icon)
  - Align Center (≡ icon centered)
  - Align Right (⫸ icon)
  - Justify (≡ icon spread)
- **Default**: Left for LTR languages, Right for RTL
- **Keyboard Shortcuts**:
  - Cmd/Ctrl + Shift + L: Left
  - Cmd/Ctrl + Shift + E: Center (E for equidistant)
  - Cmd/Ctrl + Shift + R: Right
  - Cmd/Ctrl + Shift + J: Justify
- **Toggle Behavior**: Clicking active alignment has no effect
- **Visual**: Text realigns immediately on canvas

#### **Vertical Alignment** (for text boxes with fixed height)
- **Control Type**: Three icon buttons
- **Options**:
  - Top (⫯ icon)
  - Middle (≡ icon with center highlight)
  - Bottom (⫱ icon)
- **Default**: Top
- **Availability**: Only enabled for text boxes with defined height (not auto-height)
- **Use Case**: Centering text in fixed containers

#### **Text Direction**
- **Control Type**: Dropdown
- **Options**:
  - Left-to-Right (LTR)
  - Right-to-Left (RTL)
  - Vertical Right-to-Left
  - Vertical Left-to-Right
- **Default**: Based on document language setting
- **Effect**: Changes reading direction and alignment defaults
- **Dependencies**: Affects cursor movement and selection behavior

#### **Indentation**
- **Control Type**: Three numeric inputs
- **Fields**:
  - **Left Indent**: 0-500px, default 0
  - **Right Indent**: 0-500px, default 0
  - **First Line Indent**: -500 to 500px, default 0 (negative for hanging indent)
- **Unit**: px (can switch to % of text box width)
- **Visual**: Ruler appears on canvas showing indent markers (draggable)
- **Presets**: Standard (0), Paragraph (20px first line), Hanging (-20px first line, 20px left)

#### **Text Box Dimensions**
- **Control Type**: Numeric inputs with lock aspect ratio toggle
- **Fields**:
  - **Width**: Auto or 1-10000px
  - **Height**: Auto or 1-10000px
  - **Lock icon**: Maintains aspect ratio when enabled
- **Default**: Auto for both (expands with content)
- **Auto Behaviors**:
  - Auto Width: Expands horizontally, no wrapping (for single-line text)
  - Auto Height: Wraps text, expands vertically
  - Fixed Width + Auto Height: Most common for paragraphs
- **Units**: px, %, vh/vw (viewport units)
- **Constraints**: Respect canvas boundaries with warning if oversized

#### **Position (X, Y Coordinates)**
- **Control Type**: Two numeric inputs
- **X Position**: 0 to canvas width
- **Y Position**: 0 to canvas height
- **Origin Point Selector**: 9-point grid showing which corner/edge/center is being positioned
- **Unit**: px by default, mm/cm/in for print documents
- **Precision**: 0.1px with decimal entry
- **Keyboard**: Arrow keys nudge by 1px, Shift+arrow by 10px
- **Smart Guides**: Enable snapping to guides, other elements, canvas edges

---

### **Section 4: Color and Effects**

#### **Text Color**
- **Control Type**: Color picker with multiple input methods
- **Interface**:
  - **Swatch**: Click to open full picker
  - **Hex Input**: #000000 format
  - **RGB Inputs**: 0-255 for each channel
  - **HSL Inputs**: Alternative color model
  - **Opacity Slider**: 0-100%
- **Default**: #000000 (black)
- **Features**:
  - **Eyedropper Tool**: Sample color from anywhere on screen
  - **Recent Colors**: Last 10 used colors
  - **Document Colors**: Colors currently used in document
  - **Saved Swatches**: User-saved color palette
  - **Gradient Option**: Opens gradient editor for gradient text fills
- **Accessibility**: Color contrast checker against background (WCAG AA/AAA indicators)
- **Keyboard**: Tab through recent colors, Enter to select

#### **Background Color** (text box fill)
- **Control Type**: Color picker (identical to text color)
- **Default**: None (transparent)
- **Special Option**: "None" checkbox for transparent background
- **Opacity**: Independent opacity control
- **Use Case**: Highlighting text, creating text blocks with background

#### **Stroke (Outline)**
- **Control Type**: Expandable subsection
- **Enable Toggle**: Checkbox to enable stroke
- **Stroke Color**: Full color picker
- **Stroke Width**: 0.1-50px slider + input
- **Stroke Position**: 
  - Outside (default)
  - Center
  - Inside
- **Stroke Style**:
  - Solid (default)
  - Dashed (customizable dash pattern)
  - Dotted
- **Default**: Off (no stroke)
- **Use Case**: Outlined text effects, emphasis

#### **Shadow**
- **Control Type**: Expandable subsection with enable toggle
- **Properties**:
  - **Offset X**: -100 to 100px, default 2px
  - **Offset Y**: -100 to 100px, default 2px
  - **Blur Radius**: 0-50px, default 4px
  - **Spread**: -50 to 50px, default 0px
  - **Color**: Full color picker with opacity
  - **Shadow Type**: Drop shadow vs. Inner shadow toggle
- **Presets**: Subtle, Medium, Dramatic (one-click application)
- **Default**: Off
- **Visual**: Live preview on canvas while adjusting
- **Multiple Shadows**: "+" button to add additional shadow layers

#### **Text Effects** (Advanced)
- **Control Type**: Expandable section with effect library
- **Available Effects**:
  - **3D Extrusion**: Depth, angle, lighting
  - **Neon Glow**: Glow color, intensity, spread
  - **Emboss**: Depth, direction
  - **Outline**: Multiple outline layers
  - **Pattern Fill**: Texture/pattern overlay
  - **Gradient Fill**: Full gradient editor
- **Apply**: Dropdown or modal for complex effects
- **Stack**: Multiple effects can be layered
- **Order**: Drag to reorder effect layers
- **Default**: None

### **Section 5: List Properties** (only visible for list elements)

#### **List Type**
- **Control Type**: Dropdown with visual previews
- **Options**:
  - Bulleted (various bullet styles)
  - Numbered (various number formats)
  - Lettered
  - Roman numerals
  - Custom character
  - Checklist
- **Default**: Based on element type selected from left panel
- **Change**: Converts existing list to new type

#### **Bullet/Number Style**
- **Control Type**: Visual style picker
- **For Bullets**:
  - Solid circle •
  - Empty circle ○
  - Solid square ▪
  - Empty square □
  - Diamond ◆
  - Arrow →
  - Checkmark ✓
  - Custom character (text input)
  - Custom image (upload small icon)
- **For Numbers**:
  - 1. 2. 3.
  - 1) 2) 3)
  - (1) (2) (3)
  - 01. 02. 03. (zero-padded)
- **For Letters**:
  - A. B. C.
  - a. b. c.
  - (A) (B) (C)
  - (a) (b) (c)
- **Size**: Bullet/number size relative to text (50-150%, default 100%)
- **Color**: Independent color from text (inherits by default)

#### **List Indentation**
- **Control Type**: Numeric input
- **Hanging Indent**: Distance from bullet to text start
- **Default**: 20px
- **Range**: 0-200px
- **Nested List Indent**: Additional indent per level (default 30px)

#### **List Spacing**
- **Control Type**: Numeric input
- **Between Items**: Vertical space between list items
- **Default**: 8px
- **Range**: 0-50px
- **Before List**: Space before first item
- **After List**: Space after last item

#### **Start Number** (for numbered lists)
- **Control Type**: Numeric input
- **Default**: 1
- **Range**: Any integer (including negative)
- **Use Case**: Continuing lists, custom sequences

#### **Nested List Behavior**
- **Control Type**: Checkbox options

## 4. MOUSE INTERACTION PATTERNS

Comprehensive documentation of every mouse-based interaction with text elements.

### **Selection Behaviors**

#### **Single Click: Unselected Text Element**
**Action**: Click anywhere on text element
**Result**:
- Text element becomes selected
- Bounding box appears with 8 resize handles
- Rotation handle appears above bounding box
- Right properties panel populates with text properties
- Contextual ribbon appears with text tools
- Smart alignment guides activate
- Element highlights with selection color (default blue)
- Other selected elements deselect (unless Shift held)

**Visual Feedback**:
- Bounding box: 1px solid blue line
- Handles: 8px × 8px white squares with blue border
- Rotation handle: Circular icon 20px above top-center
- Cursor: Default arrow (not text cursor yet)

**Edge Cases**:
- Click on transparent part of text box: Selects text box
- Click directly on character: Still selects element (doesn't enter edit mode)
- Click on locked element: Shows "locked" indicator, no selection

---

#### **Single Click: Already Selected Text Element**
**Action**: Click on already-selected text element
**Result** (depends on text type):

**For Editable Text Boxes** (paragraphs, body text):
- Enters text editing mode
- Text cursor (I-beam) appears at click position
- Bounding box changes to edit-mode styling (lighter color)
- Selection handles hidden or dimmed
- Contextual ribbon switches to editing controls
- Character-level formatting becomes available
- Click position determines cursor placement between characters

**For Artistic Text** (single-line, effects):
- Enters inline editing mode
- Cursor appears at click position
- Can edit text content directly
- Effects remain visible during editing

**For Special Text** (dates, page numbers, merge fields):
- May show field editor or
- Enter text edit mode with field highlighted
- Depends on field type

**Visual Feedback**:
- Blinking text cursor (1px line, 500ms blink rate)
- Bounding box color changes (blue → light gray)
- Text becomes editable inline

---

#### **Double Click: Unselected Text Element**
**Action**: Double-click on text element
**Result**:
- Skips selection state, goes directly to editing mode
- Text cursor positioned at double-click location
- Additionally: Selects the word at click position (word selection)
- Word boundaries determined by spaces and punctuation
- Triple-click for paragraph selection

**Use Case**: Fast path to editing without intermediate selection step

---

#### **Triple Click: Text Element**
**Action**: Triple-click on text
**Result**:
- Enters edit mode AND
- Selects entire paragraph (or entire text for single-line elements)
- Allows immediate replacement or formatting of all text
- Quadruple-click selects entire text element content

**Visual**: Entire paragraph highlighted with selection color

---

#### **Click and Drag: Creating New Text Element**
**Action**: Select text tool from left panel, then click and drag on canvas
**Result**:
- First click sets top-left corner anchor point
- Drag defines width and height of text box
- Live preview shows rectangle outline as you drag
- Dimensions tooltip follows cursor (e.g., "450 × 200 px")
- Snap guides appear at alignment points
- Release completes text box creation
- Automatically enters edit mode with cursor ready
- Default placeholder text may appear (e.g., "Type something")

**Visual Feedback**:
- Dashed rectangle outline during drag
- Dimensions tooltip in corner
- Snap lines when near guides/elements
- Cursor shows text tool icon while dragging

**Constraints**:
- Minimum size: 20px × 20px (prevents accidental tiny boxes)
- Respects canvas boundaries (can't drag beyond)
- Grid snapping if enabled (to 10px or custom grid)
- Aspect ratio lock: Hold Shift for square text box

**Single Click** (not drag):
- Creates auto-width text element at click point
- Expands as you type (no predefined width)
- Use case: Single-line headlines, labels

---

#### **Shift + Click: Multi-Selection**
**Action**: Hold Shift, click on additional text elements
**Result**:
- Adds clicked element to current selection
- All selected elements show bounding boxes
- Shared properties appear in right panel (with "Mixed" for different values)
- If clicking already-selected element: Deselects it (toggle behavior)
- Selection count shown in status bar

**Multi-Selection Bounding Box**:
- Group bounding box encompasses all selected elements
- Single set of transform handles for the group
- Transformations apply to all elements simultaneously

**Visual**: Each element outlined, group bounding box in different color (e.g., purple)

---

#### **Command/Ctrl + Click: Add/Remove from Selection**
**Action**: Cmd/Ctrl + Click on element
**Result**: Same as Shift+Click (platform-specific equivalent)

**Additional Behavior**:
- Click on grouped element: Selects individual element within group
- Cmd/Ctrl + Click again: Selects parent group
- Progressive selection drilling (element → group → parent group)

---

#### **Alt/Option + Click: Select Through Layers**
**Action**: Alt + Click on overlapping text elements
**Result**:
- Cycles through stacked elements at click position
- Selects next element in Z-order (layering)
- Visual indication of available elements (temporary outline flashes)
- Tooltip shows element name during cycling
- Use case: Selecting hidden or obscured text

**Visual**: Brief highlight (100ms) on element about to be selected

---

### **Dragging and Transformation**

#### **Drag Selected Text Element: Move**
**Action**: Click and drag on selected text element (not on handles)
**Result**:
- Element moves with cursor
- Position updates in real-time
- Coordinate tooltip shows X, Y position
- Smart guides appear when aligning with:
  - Other elements' edges
  - Canvas center
  - Gridlines
  - Custom guides
- Snap behavior: Elements snap to guides (8px tolerance)
- Magnetic snap indicator (visual and optional haptic feedback)

**Visual Feedback**:
- Semi-transparent ghost shows original position
- Smart guide lines (magenta) appear at alignment points
- Distance measurements shown between nearby elements
- Cursor changes to four-directional move arrow

**Modifiers**:
- **Shift**: Constrains movement to horizontal or vertical (whichever direction moved first)
- **Alt/Option**: Duplicates while dragging (creates copy, original stays)
- **Cmd/Ctrl**: Temporarily disables snapping for precise placement

**Multi-Element Drag**:
- All selected elements move together
- Relative positions maintained
- Snap guides based on group bounding box

---

#### **Corner Handle Drag: Proportional Resize**
**Action**: Drag any corner handle
**Result**:
- Resizes text box proportionally from opposite corner (anchor point)
- Maintains aspect ratio
- Text reflows within new dimensions
- Dimension tooltip shows width × height
- Text may change line breaks as width adjusts

**Visual**:
- Opposite corner stays fixed (anchor point indicator)
- Resize preview shows new boundary
- Text reflows in real-time (performance permitting)

**Modifiers**:
- **Shift**: Maintains aspect ratio more strictly (removes diagonal drift)
- **Alt/Option**: Resizes from center instead of opposite corner
- **Cmd/Ctrl**: Free resize (doesn't maintain aspect ratio) - can distort

**Font Scaling**:
- By default: Text reflows, font size unchanged
- With Cmd+Shift held: Scales font size proportionally (maintains visual appearance)
- Preference setting to change default behavior

---

#### **Side Handle Drag: Single-Dimension Resize**
**Action**: Drag top, bottom, left, or right handle
**Result**:
- Stretches text box in one direction only
- Opposite side stays fixed
- **Horizontal handles** (left/right): Changes width, text reflows
- **Vertical handles** (top/bottom): Changes height, may clip or add space

**Visual**: Dimension guide shows changing measurement

**Text Behavior**:
- Width change: Reflows text, adjusts line breaks
- Height change with overflow-hidden: May clip text
- Height change with auto-height: Resizes to fit content

**Modifiers**:
- **Shift**: Maintains opposite side at fixed position (default behavior)
- **Alt/Option**: Resizes symmetrically from center

---

#### **Rotation Handle Drag: Rotate**
**Action**: Drag circular rotation handle above element
**Result**:
- Text element rotates around its center point (or custom pivot)
- Rotation angle shown in tooltip (e.g., "45°")
- Snaps to 15° increments (customizable)
- Bounding box rotates with text
- Text content rotates as visual element

**Visual Feedback**:
- Rotation angle arc visualization
- Angle measurement tooltip
- Ghost outline at 0° (original orientation)
- Snap indicators at 45°, 90°, 180°, 270°

**Modifiers**:
- **Shift**: Snaps to 15° increments (or 45° if preference set)
- **Cmd/Ctrl**: Free rotation (no snapping)
- **Alt/Option**: Duplicates and rotates copy

**Pivot Point**:
- Default: Center of text element
- Click and drag pivot point indicator to reposition (appears during rotation)
- Common positions: 9-point grid on bounding box

---

#### **Special Handle: Rounded Corners** (for text boxes with borders)
**Action**: Drag diamond-shaped handle that appears on text boxes with borders
**Result**:
- Adjusts corner radius of text box
- All corners adjust uniformly by default
- Radius value shown in tooltip (e.g., "12px")

**Modifiers**:
- **Alt/Option**: Adjust only the closest corner (individual corner control)
- **Shift**: Snap to common values (0px, 5px, 10px, 15px, 20px)

---

#### **Special Handle: Artistic Text Letter Spacing**
**Action**: For artistic text, horizontal handle appears between letters
**Result**:
- Adjusts spacing between specific letter pairs (kerning)
- Manual kerning override
- Value in em units or pixels shown

---

#### **Special Handle: Text on Path Controls**
**Action**: Text on curved path shows path control handles
**Result**:
- Bezier curve handles to reshape text path
- Drag handle to adjust curve
- Text follows path in real-time
- Additional handle to slide text along path (start position)

### **Modal 6.2: Find and Replace**

**Trigger**:
- Menu: Edit → Find and Replace
- Keyboard: Cmd/Ctrl+F (simple find) or Cmd/Ctrl+H (find and replace)
- Toolbar: Search icon

**Purpose**: Search text content across selected elements, all elements, or entire document

**Dialog Layout**:
- **Size**: 600px × 400px, non-resizable initially, can expand
- **Position**: Top-right of screen (doesn't obscure text)
- **Modeless**: Can interact with canvas while dialog is open

**Interface**:

**Find Section**:
- **Find What**: Text input field, full keyboard focus on open
- **Search In**:
  - Current Text Element (default if one selected)
  - All Text Elements on Current Page
  - All Text Elements in Document
  - Selected Text Only (if text highlighted)
- **Options** (checkboxes):
  - Match Case (Aa)
  - Match Whole Words Only
  - Use Regular Expressions (regex)
  - Search Backwards
  - Wrap Around (loop to beginning)

**Replace Section** (expandable):
- **Replace With**: Text input field
- **Preserve Formatting**: Checkbox (keeps original formatting when replacing)

**Action Buttons**:
- **Find Next**: Highlights next match, scrolls to view
- **Find Previous**: Highlights previous match
- **Find All**: Selects all matches, shows count
- **Replace**: Replaces current match, moves to next
- **Replace All**: Replaces all matches (with confirmation)
- **Count**: Shows number of matches without selecting

**Results Panel** (bottom half, appears after search):
- **List of Matches**: Shows each match with context
  - Element name/type
  - Surrounding text (10 words before/after)
  - Page number if multi-page
  - Click to jump to that match
- **Match Count**: "X matches found" at bottom
- **Progress Bar**: For large documents during "Find All"

**Highlighting**:
- Current match: Bright yellow highlight
- Other matches: Pale yellow
- Matches scroll into view automatically
- Highlight persists until "Clear" button pressed

**Advanced Options** (collapsible section):
- **Scope**: Layer, Group, or Document

### **Key Insights from the Matrix**:

1. **Most Frequently Used Properties** (Font, Size, Color, Alignment) are accessible in:
   - Right Panel (persistent, detailed)
   - Contextual Ribbon (quick access, visual)
   - Keyboard shortcuts (fastest for power users)

2. **Advanced Properties** (OpenType, Hyphenation, Complex Effects) primarily accessed via:
   - Modal dialogs (focused, comprehensive interface)
   - Right Panel advanced sections (for iterative work)

3. **Contextual Ribbon** focuses on:
   - Properties that change frequently during editing
   - Visual/icon-based controls for quick identification
   - Most common text formatting tasks

4. **Right Panel** is:
   - Comprehensive (nearly all properties accessible)
   - Organized for browsing and discovery
   - Persistent (always available when element selected)

5. **Keyboard Shortcuts** cover:
   - High-frequency operations (Bold, Italic, Alignment)
   - Navigation and selection
   - Quick formatting changes
   - NOT granular property value setting (e.g., specific font size)

6. **Context Menus** provide:
   - Quick access to common actions
   - Clipboard operations
   - Arrangement and transformation
   - Entry points to modals/panels
   - NOT detailed property editing

7. **Hover/Inline Controls** used for:
   - Direct manipulation (dragging handles)
   - Visual affordances (indicating interactive elements)
   - Quick previews
   - NOT primary property setting

8. **Redundancy is Intentional**:
   - Different users have different workflows
   - Efficiency varies by task context
   - Discoverability vs. expert efficiency balanced
   - No single "right" way to access properties

## 8. VISUAL FEEDBACK AND STATE COMMUNICATION

Every interaction and state change must be communicated clearly to the user through immediate, unambiguous visual feedback.

### **8.1 Selection States**

#### **Unselected Element**
- **Appearance**: Element renders normally, no additional visual indicators
- **Hover State**: 
  - 1px light blue outline (#60A5FA, 50% opacity) appears around text bounds
  - Cursor changes to pointer/hand (indicating clickable)
  - Subtle highlight/glow effect (optional, preference-based)
  - Layer name tooltip after 800ms delay
  - Quick action icons may fade in near corners (Edit, Duplicate, Delete)

#### **Selected Element (Single Selection)**
- **Bounding Box**: 1px solid blue line (#3B82F6) around text element
- **Resize Handles**: 8 handles (4 corners, 4 sides)
  - **Appearance**: 8px × 8px white squares with 2px blue border
  - **Corner handles**: Diagonal resize (two-way arrows cursor)
  - **Side handles**: Single-direction resize (horizontal/vertical arrows cursor)
  - **Hover on handle**: Handle increases to 10px × 10px, brighter blue (#60A5FA)
- **Rotation Handle**: 
  - **Position**: 24px above top-center of bounding box
  - **Appearance**: Circular icon (rotation arrow), 16px diameter
  - **Connector**: Thin blue line from rotation handle to bounding box
  - **Hover**: Enlarges to 20px, cursor shows rotation icon
- **Additional Indicators**:
  - If text has links: Small chain-link icon in corner
  - If text is locked: Red padlock icon overlay
  - If text overflows (hidden): Red "+" icon at overflow point
  - If text is part of group: Subtle purple tint on bounding box
- **Right Panel**: Populates with all text properties
- **Contextual Ribbon**: Appears above canvas with text tools
- **Status Bar** (bottom of app): Shows element type, size, position

#### **Selected Element (Multi-Selection)**
- **Individual Elements**: Each shows lighter blue outline (not full bounding box)
- **Group Bounding Box**: Encompasses all selected elements
  - **Color**: Purple (#9333EA) to differentiate from single selection
  - **Handles**: Same 8 resize handles, but operate on group
  - **Transformation**: Affects all selected elements simultaneously
- **Right Panel**: 
  - Shows properties common to all elements
  - "Mixed" indicator where values differ (e.g., "Mixed" in font size field if elements have different sizes)
  - Changing property applies to all
- **Contextual Ribbon**: Shows common tools, grays out incompatible options
- **Count Indicator**: Badge showing "3 elements selected" in status bar or ribbon

#### **Editing Mode (Text Cursor Active)**
- **Bounding Box**: Changes from solid blue to light gray (#E5E7EB), less prominent
- **Resize Handles**: Hidden or significantly dimmed (prevents accidental dragging while editing)
- **Text Cursor**: 
  - **Appearance**: 1px vertical line, height matching current line height
  - **Color**: Blue (#3B82F6) or contrasting color against background
  - **Blink Rate**: 500ms on, 500ms off
  - **Position**: Between characters, indicating insertion point
- **Selection Highlight** (when text selected):
  - **Background**: Light blue (#DBEAFE) highlight behind selected text
  - **Text**: Remains in original color (not inverted, for readability)
  - **Multi-line**: Highlight follows line breaks, shows across all lines
- **Contextual Ribbon**: Switches to editing tools (character/paragraph formatting)
- **Right Panel**: Shows properties of selected text (or cursor position if no selection)
- **Cursor Shape**: I-beam when over text area

#### **Locked Element**
- **Visual Indicator**: 
  - Red padlock icon overlay (16px, semi-transparent, in top-right corner)
  - Bounding box color changes to red (#EF4444) when attempting interaction
  - Cursor shows "prohibited" symbol (circle with slash) on hover
- **Interaction**: 
  - No dragging or resizing allowed
  - Click shows tooltip: "Element is locked. Unlock to edit."
  - Context menu shows "Unlock" as primary option
- **Layer Panel**: Lock icon next to element name

#### **Hidden Element**
- **Canvas**: Element not visible
- **Layer Panel**: Element name appears in light gray italic, eye icon shown crossed out
- **Selection**: Cannot be selected from canvas, can be selected from layers panel
- **Reveal**: "Show All Hidden" command (Cmd+Opt+Shift+H) or eye icon click in layers

### **8.2 Hover States**

#### **Hover Over Element (Not Selected)**
- **Outline**: 1px dashed light blue (#93C5FD) outline appears
- **Cursor**: Changes to default pointer (arrow)
- **Tooltip**: After 800ms, shows:
  - Element type: "Body Text" or "Heading 1"
  - First 30 characters of content: "Lorem ipsum dolor sit..."
  - Dimensions: "450 × 120 px"
  - Layer name if custom: "Footer Info"
- **Quick Actions** (optional, based on preferences):
  - Small icons fade in near element corners
  - Edit icon (pencil)
  - Duplicate icon (two overlapping squares)
  - Delete icon (trash)
  - Hover over icon shows action name

#### **Hover Over Resize Handles**
- **Handle**: Enlarges from 8px to 10px
- **Color**: Brightens to #60A5FA
- **Cursor**: Changes to appropriate directional arrow
  - **Corner handles**: ↖↘ or ↙↗ (diagonal two-way arrows)
  - **Side handles**: ↔ (horizontal) or ↕ (vertical)
  - **Rotation handle**: ↻ (circular rotation cursor)
- **Tooltip** (after 500ms): "Resize width and height" or "Rotate element"

#### **Hover Over Link (in Edit Mode)**
- **Cursor**: Changes to hand/pointer icon
- **Underline**: Becomes more prominent (if already underlined) or appears
- **Tooltip**: Shows full URL (e.g., "https://example.com/page") after 600ms
- **Color Shift**: Link color slightly brighter (visual affordance)
- **Ctrl/Cmd Modifier**: Tooltip shows "Ctrl+Click to follow link"

#### **Hover Over Misspelled Word (in Edit Mode)**
- **Wavy Underline**: Red, remains visible
- **Tooltip** (after 400ms): 
  - Shows first 3 spelling suggestions
  - "Add to Dictionary" option
  - "Ignore" option
- **Right-click**: Opens full spell check context menu

### **8.3 Active/Interaction States**

#### **Dragging Element (Move)**
- **Original Position**: 
  - Semi-transparent ghost outline remains (30% opacity)
  - Shows where element started
- **Moving Element**:
  - Full opacity, follows cursor
  - Slight drop shadow (2px offset, 4px blur, 30% opacity) for depth
- **Cursor**: Four-directional move arrows (⬍)
- **Coordinate Tooltip**:
  - Follows cursor, offset by 20px down-right
  - Shows current X, Y position: "X: 245 Y: 680"
  - Updates in real-time as dragging
- **Smart Guides**:
  - **Appearance**: Bright magenta (#EC4899) lines, 1px, extend across canvas
  - **Trigger**: When element edge aligns within 8px of:
    - Other elements' edges
    - Canvas center (horizontal/vertical)
    - Gridlines (if visible)
    - Custom guides
  - **Snap Behavior**: Element "magnetically" snaps to guide when within 8px tolerance
  - **Visual Snap**: Brief highlight pulse on guide line when snap occurs
  - **Distance Markers**: Small tooltip showing distance between elements (e.g., "20px")
- **Multiple Elements**: All move together, maintaining relative positions

#### **Resizing Element (Handle Drag)**
- **Bounding Box**: Updates in real-time to show new size
- **Handle**: Remains under cursor (sticky)
- **Ghost Outline**: Semi-transparent outline shows original size
- **Dimension Tooltip**:
  - Positioned near cursor
  - Shows width × height: "450 × 200 px"
  - Updates live during drag
  - If maintaining aspect ratio: Shows lock icon ⚿
- **Text Reflow**: 
  - Text reflows within new dimensions in real-time
  - May show performance-optimized preview if complex formatting
  - Line breaks adjust dynamically
- **Constraints**:
  - If Shift held: Maintains aspect ratio, tooltip shows lock icon
  - If Alt/Opt held: Resizes from center, both tooltip and visual center indicator
  - Snap to grid (if enabled): Dimensions snap to grid increments

#### **Rotating Element (Rotation Handle Drag)**
- **Rotation Arc**: 
  - Semi-transparent arc drawn from center to rotation handle
  - Color: Light blue (#93C5FD), 2px line
  - Radius: Distance from center to rotation handle
- **Angle Measurement**:
  - Tooltip shows angle: "45°" or "127°"
  - Updates in real-time
  - Positioned near rotation handle
- **Snap Indicators**:
  - At 15° increments (if enabled): Subtle tick marks on arc
  - At major angles (0°, 45°, 90°, 135°, 180°, 270°): Larger tick marks and brief snap pulse
  - Magnetic snap: Rotation "sticks" briefly at snap angles
- **Pivot Point**: 
  - Visible crosshair at center (default)
  - Can be dragged to new position before rotating
  - Pivot crosshair color: Orange (#FB923C)
- **Ghost Outline**: Original orientation shown as semi-transparent outline

#### **Text Selection (Highlighting Text)**
- **Highlight Color**: Light blue (#DBEAFE) background
- **Text Color**: Unchanged (not inverted) for accessibility
- **Selection Bounds**: Follows exact character boundaries
- **Multi-line**: Highlight background extends across lines
- **Cursor**: I-beam while selecting
- **Selection Handles** (touch/mobile, optional on desktop):
  - Small circular handles at start and end of selection
  - Draggable to extend/contract selection

#### **Dragging Text Selection (Move Text)**
- **Selected Text**: Appears to lift with cursor, semi-transparent
- **Original Location**: Ghosted outline of text remains
- **Drop Indicator**: Vertical cursor line shows where text will be inserted
- **Cursor**: Changes to move cursor when dragging

#### **Copy/Paste Operation**
- **Copy**: 
  - Brief visual flash or scale animation on copied element (100ms)
  - Status message: "Copied" appears briefly near selection
- **Paste**:
  - Pasted element appears with selection outline

## 9. ACCESSIBILITY AND INCLUSIVE DESIGN CONSIDERATIONS

Ensuring all users can effectively interact with text elements regardless of abilities or assistive technologies.

### **9.1 Keyboard Navigation**

#### **Complete Keyboard Access**
- **Tab Order**: Logical sequence through all controls
  - Left panel → Canvas → Right panel → Contextual ribbon
  - Within panels: Top to bottom, left to right
  - Skip links: Jump to main content areas (Shift+Tab)
- **Focus Indicators**: 
  - Visible 2px blue outline (#3B82F6) around focused element
  - High contrast in all color modes
  - Never hidden (CSS outline-style: auto respected)
- **Enter/Return**: 
  - Activates buttons
  - Opens dropdowns
  - Submits forms
  - Enters edit mode on selected text element
- **Escape**:
  - Closes modals, dropdowns, popovers
  - Exits edit mode
  - Cancels drag operations
  - Returns to previous context
- **Arrow Keys**:
  - Navigate through dropdown options
  - Move between radio buttons
  - Navigate text cursor in edit mode
  - Nudge selected element (when not editing)
- **Space Bar**:
  - Toggles checkboxes
  - Activates buttons (alternative to Enter)
  - Scrolls canvas if no element selected

#### **Keyboard Shortcuts Accessibility**
- **Discoverable**:
  - All shortcuts shown in tooltips
  - Menu items display shortcuts
  - Help → Keyboard Shortcuts reference panel
- **Customizable**:
  - Users can remap any shortcut
  - Export/import shortcut configurations
  - Reset to defaults option
- **Conflicts Avoided**:
  - No conflicts with browser/OS shortcuts
  - Users warned if custom shortcut conflicts
- **Mnemonics**: 
  - Menu items have underlined letters for Alt+key access (Windows)
  - Logical associations (Cmd+B for Bold, Cmd+I for Italic)

#### **Focus Management**
- **Modal Dialogs**:
  - Focus moves to dialog when opened
  - Focus trapped within dialog (Tab doesn't escape)
  - First interactive element automatically focused
  - Escape returns focus to trigger element
- **Popovers/Dropdowns**:
  - Focus moves to first option when opened
  - Arrow keys navigate
  - Enter selects
  - Escape closes and returns focus
- **Canvas Interaction**:
  - Tab selects first/next element on canvas
  - Enter enters edit mode
  - Escape returns to selection mode
  - Arrow keys navigate between elements

### **9.2 Screen Reader Support**

#### **Semantic HTML and ARIA**
- **Landmarks**: 
  - `<nav>` for left panel element library
  - `<main>` for canvas area
  - `<aside>` for right panel properties
  - `<form>` for settings dialogs
- **Headings**: Logical hierarchy (h1 → h2 → h3) for screen reader navigation
- **Labels**: Every input has associated `<label>` or `aria-label`
- **Buttons**: 
  - Icon buttons have `aria-label` (e.g., `aria-label="Bold"`)
  - Disabled state communicated (`aria-disabled="true"`)
- **Live Regions**: 
  - Status messages announced (`role="status"`)
  - Errors announced immediately (`role="alert"`)
  - Progress updates (`role="progressbar"` with `aria-valuenow`)

#### **Descriptive Text**
- **Image Buttons**: Alt text or aria-label describes function, not appearance
  - Good: "Increase font size"
  - Bad: "Plus icon"
- **Contextual Information**:
  - Selected element announced: "Heading 1 text element selected"
  - Properties panel: "Font size: 48 pixels"
  - Editing mode: "Editing text: Type your content here"
- **State Changes Announced**:
  - "Bold applied"
  - "Text aligned center"
  - "Font changed to Arial"
  - "3 elements selected"

#### **Complex Widgets**
- **Custom Dropdowns**: 
  - `role="combobox"` on trigger
  - `role="listbox"` on options
  - `aria-expanded` indicates open/closed state
  - `aria-activedescendant` tracks current option
- **Sliders**: 
  - `role="slider"`
  - `aria-valuemin`, `aria-valuemax`, `aria-valuenow`
  - `aria-valuetext` for non-numeric values
- **Tabs**: 
  - `role="tablist"`, `role="tab"`, `role="tabpanel"`
  - `aria-selected` indicates active tab
  - Arrow keys navigate tabs

#### **Error Messages**
- **Immediate Announcement**: Errors announced as soon as they occur
- **Clear Language**: "Email address is invalid. Please include an @ symbol."
- **Associated with Input**: `aria-describedby` links error to input field
- **Persistent**: Error remains visible/announced until fixed

### **9.3 Visual Accessibility**

#### **Color Contrast**
- **WCAG AA Compliance** (minimum):
  - 4.5:1 for normal text (< 18pt)
  - 3:1 for large text (≥ 18pt or 14pt bold)
  - 3:1 for UI components and graphic elements
- **WCAG AAA Compliance** (preferred):
  - 7:1 for normal text
  - 4.5:1 for large text
- **Contrast Checker**:
  - Built into color picker
  - Shows AA/AAA compliance indicators
  - Suggests nearest compliant color
- **Color Independence**:
  - Never rely solely on color to convey information
  - Use icons, text labels, patterns in addition to color
  - Example: Error states use red color + "X" icon + text message

#### **High Contrast Mode**
- **Support**: Respects OS high contrast settings (Windows High Contrast, macOS Increase Contrast)
- **Custom Mode**: Built-in high contrast theme option
  - Black background, white text
  - Yellow highlights for selection
  - Bold outlines on elements
- **Overrides**: User can force high contrast regardless of theme

#### **Dark Mode**
- **Full Support**: Every UI component has dark variant
- **Maintained Contrast**: Contrast ratios preserved in dark mode
- **Auto-Detect**: Respects OS preference (prefers-color-scheme: dark)
- **Toggle**: Manual dark/light/auto mode switcher in preferences
- **Canvas**: Option for dark canvas background for reducing eye strain

#### **Text Sizing**
- **Resizable UI Text**:
  - UI respects browser/OS text size settings
  - User can increase UI text size independently (100%-200%)
  - Layout reflows to accommodate larger text
  - Minimum font size: 12px in UI (16px recommended)
- **Canvas Text**: 
  - Not affected by UI text scaling (canvas content is design output)
  - But user can zoom canvas (Cmd/Ctrl +/-) for better visibility

#### **Visual Clarity**
- **Spacing**: 
  - Generous padding between clickable elements (minimum 8px)
  - Line height for text at least 1.5× font size
  - Paragraph spacing to clearly separate blocks
- **Icons**:
  - Simple, recognizable shapes
  - Avoid overly complex or abstract icons
  - Paired with text labels where possible
- **Animations**:
  - Respect `prefers-reduced-motion` for users with vestibular disorders
  - Option to disable animations entirely in preferences
  - No auto-playing animations by default

### **9.4 Motor Accessibility**

#### **Large Click Targets**
- **Minimum Size**: 44px × 44px for all interactive elements (WCAG 2.1 AAA)
- **Spacing**: Minimum 8px between adjacent targets to prevent mis-clicks
- **Hover Tolerance**: 
  - Hover states have slight buffer (1-2px expansion)
  - Doesn't require pixel-perfect precision

#### **Drag and Drop Alternatives**
- **Every Drag Operation Has Keyboard Equivalent**:
  - Drag to move element → Arrow keys to nudge
  - Drag to resize → Numeric input in properties
  - Drag to reorder → Up/Down arrow buttons
- **Click-to-Place**: 
  - Alternative to click-and-drag for creating elements
  - Click sets position, element created at default size
  - Modify size via properties panel

#### **Forgiving Interactions**
- **Cancellable**: 
  - Escape key cancels drags
  - Moving cursor off before release cancels click
- **Undo Available**: 
  - All operations undoable (Cmd+Z)
  - Multi-level undo (50+ steps)
- **No Time Limits**: 
  - No operations require speed or dexterity
  - Tooltips/menus stay open until dismissed (no auto-hide on timer)

#### **Sticky Keys & Modifier Support**
- **Sequential Input**: 
  - Supports "sticky keys" for users who can't hold multiple keys
  - Pressing Shift, then separately pressing another key works
- **Alternative Modifiers**: 
  - Where Cmd+Click is required, provide alternative without modifier
  - Context menus accessible via keyboard

### **9.5 Cognitive Accessibility**

#### **Clear Language**
- **Simple Terms**: 
  - "Make bold" instead of "Increase font weight to 700"
  - "Align left" instead of "Set text-align property to start"
- **Consistent Terminology**: 
  - Same action always called the same thing
  - "Delete" not sometimes "Remove" or "Erase"
- **Avoid Jargon**: 
  - Explain technical terms when necessary
  - Tooltips provide context for unfamiliar concepts

#### **Visual Hierarchy**
- **Importance Indicated by**:
  - Size (larger = more important)
  - Color (bright/saturated = action items)
  - Position (top-left = primary)
- **Grouping**: Related controls visually grouped with borders or spacing
- **Progressive Disclosure**: 
  - Advanced options hidden until needed
  - "Show more" expanders for detail

#### **Error Prevention**
- **Confirmation Dialogs**: 
  - For destructive actions (delete, clear formatting)
  - "Are you sure you want to delete this text element?"
  - Checkbox option: "Don't ask again" for experienced users
- **Default Safe Options**: 
  - Defaults prevent accidental data loss
  - Example: "Cancel" button is default focus, not "Delete"
- **Undo Always Available**: Reduces anxiety about mistakes

#### **Help and Documentation**
- **Contextual Help**: 
  - "?" icons open help for current panel/dialog
  - Tooltips explain every control
- **Onboarding**: 
  - First-time user tutorial (optional, skippable)
  - Highlights key features
- **Search**: Help documentation fully searchable
- **Examples**: Visual examples of each feature's effect

#### **Consistency**
- **Predictable Behavior**: 
  - Same interaction patterns throughout app
  - Buttons in same positions (OK on right, Cancel on left)
- **Familiar Conventions**: 
  - Follows platform conventions (macOS vs. Windows)
  - Similar to other design tools where possible

### **9.6 Assistive Technology Compatibility**

#### **Screen Readers Tested**
- **macOS**: VoiceOver
- **Windows**: JAWS, NVDA
- **Linux**: Orca
- **Mobile**: TalkBack (Android), VoiceOver (iOS)

#### **Screen Magnifiers**
- **Zoom Support**: UI remains usable at 200%-400% zoom
- **No Horizontal Scrolling**: At 200% zoom, no horizontal scroll needed
- **Reflow**: Content reflows to fit magnified view

#### **Voice Control**
- **All Actions Named**: Every button, control has unique name for voice commands
- **Grid Navigation**: Supports voice control grid overlays

#### **Switch Access**
- **Sequential Navigation**: All controls accessible via switch scanning
- **Grouped Actions**: Related controls grouped for faster navigation

## 10. ELEMENT-SPECIFIC DEEP DIVES

Advanced features unique to text elements, beyond general framework.

### **10.1 Advanced Text Path Features (Text on Path)**

#### **Path Creation and Editing**
- **Pre-Defined Paths**:
  - Circle/Arc (adjustable radius, start/end angles)
  - Wave (sine wave with frequency, amplitude controls)
  - Spiral (Archimedean or logarithmic)
  - Custom Shapes (star, polygon, etc.)
- **Custom Bezier Paths**:
  - Click to add anchor points
  - Drag handles to create curves
  - Convert point types: corner, smooth, symmetric
  - Close path or leave open
- **Import Path**: 
  - Paste SVG path data
  - Import from vector shape on canvas
- **Path Visibility**:
  - Toggle: Show path as stroke (for editing) or hide (final output)
  - Stroke color/width customizable for guides

#### **Text Positioning on Path**
- **Alignment Options**:
  - **Start**: Text begins at path start point
  - **Center**: Text centered on path
  - **End**: Text ends at path end point
  - **Justify**: Stretches text to fill entire path
- **Baseline Offset**:
  - Distance above/below path centerline
  - Range: -500px to +500px
  - Useful for creating layered effects (e.g., text on both sides of circle)
- **Start Position Slider**:
  - Slide text along path without changing alignment
  - Percentage (0-100%) of path length
- **Flip Side**:
  - Toggle text to inside vs. outside of closed path
  - Or above vs. below for open path

#### **Character Handling**
- **Rotation Options**:
  - **Tangent to Path** (default): Characters rotate to follow path curvature
  - **Upright**: Characters remain vertical, only position follows path
  - **Perpendicular**: Characters perpendicular to path tangent
- **Spacing Modes**:
  - **Auto**: Even visual spacing along path
  - **Fixed**: Consistent character spacing (may bunch on tight curves)
  - **Manual Kerning**: Adjust spacing between individual letters

#### **Advanced Path Effects**
- **Repeating Text**: Repeat text multiple times around closed path (e.g., circular badge)
- **Offset Copies**: Create concentric rings of text
- **Path Morphing**: Animate path shape (if exporting interactive/video)

### **10.2 Variable and Dynamic Text Advanced**

#### **Mail Merge Field Types**
1. **Text Fields**: 
   - Map to text columns in data source
   - Formatting: Uppercase, lowercase, title case
   - Trimming: Auto-trim whitespace
   - Fallback: Default value if empty

2. **Number Fields**:
   - Format: Decimal places (0-10)
   - Thousands separator (comma, period, space, none)
   - Negative numbers: (123), -123, or 123-
   - Prefix/Suffix: $ for currency, % for percentage

3. **Date/Time Fields**:
   - Input format detection (MM/DD/YYYY, ISO 8601, etc.)
   - Output format selection (50+ formats)
   - Locale-specific formatting
   - Relative dates: "3 days ago", "Next Tuesday"

4. **Image Fields**:
   - URL or file path in data source
   - Image fitting: Fill, fit, stretch, center
   - Fallback image if URL broken
   - Image caching for performance

5. **Boolean Fields**:
   - True/false values in data
   - Display as: Yes/No, True/False, custom text, or icons
   - Conditional visibility: Show/hide entire text block based on boolean

6. **List/Array Fields**:
   - Multiple values in single cell (comma-separated)
   - Display as: Bulleted list, numbered list, comma-separated, custom separator

#### **Conditional Logic**
- **Simple Conditionals**:
  - If [Field] = "Value", show "Text A", else show "Text B"
  - Visual rule builder (no code)
  - Comparison operators: =, ≠, >, <, ≥, ≤, contains, starts with, ends with
- **Complex Conditions**:
  - AND/OR logic: If [Field1] = "X" AND [Field2] > 100
  - Nested conditions: If (A AND B) OR (C AND D)
  - Regular expressions: If [Field] matches pattern

#### **Formulas and Calculations**
- **Basic Math**: 
  - Add, subtract, multiply, divide fields
  - Example: `{Price} × {Quantity} = {Total}`
- **Text Manipulation**:
  - Concatenate: `{FirstName} + " " + {LastName}`
  - Substring: First 10 characters of {Description}
  - Replace: Replace "LLC" with "Limited" in {CompanyName}
- **Date Calculations**:
  - Add days: `{OrderDate} + 7 days`
  - Difference: Days between {StartDate} and {EndDate}
- **Lookup/Reference**:
  - VLOOKUP-style: Find value in external table
  - Default values: If {Field} is empty, use "N/A"

#### **Data Validation**
- **Pre-Generation Validation**:
  - Check all fields have values (or allowed to be empty)
  - Validate data types (number fields contain numbers)
  - Range validation (age between 0-120)
  - Pattern matching (email format, phone format)
- **Error Handling**:
  - List all validation errors before generation
  - Highlight problematic rows in data source
  - Options: Skip invalid rows, use default values, or abort

#### **Preview and Testing**
- **Record Navigation**:
  - Previous/Next buttons to cycle through data
  - Jump to specific record number
  - Search/filter records by field value
  - Random sample: Preview random selection of records
- **Side-by-Side Comparison**: 
  - View multiple records simultaneously
  - Spot inconsistencies in data
- **Export Preview**: Generate PDF/PNG of first 5 records for review

### **10.3 List Formatting Advanced Features**

#### **Multi-Level Lists (Outline Numbering)**
- **Level Definition**:
  - Up to 9 indent levels

  - Level 1: 1, 2, 3
  - Level 2: 1.1, 1.2, 1.3
  - Level 3: 1.1.1, 1.1.2, etc.

  - Legal numbering: 1.1, 1.1.1, 1.1.1.1
  - Custom separators: 1-1, 1.1, 1:1
- **Automatic Continuation**:

  - Option to restart at each section
  - Link lists: "Continue from previous list" option

#### **Custom Bullets and Icons**
- **Image Bullets**:
  - Upload custom icon (PNG, SVG)
  - Max size: 64×64px
  - Auto-scaling to match text size
  - Color overlay option (for monochrome icons)
- **Unicode Characters**:
  - Full Unicode support for bullets
  - Emoji bullets: ✓, ★, ❤, 🔥, etc.
  - Geometric shapes: ◆, ▶, ▸, ▪
- **Dynamic Bullets**:
  - Bullet changes based on list item content
  - Example: Checkmarks for completed items, X for incomplete
  - Requires conditional formatting

#### **Smart Lists**
- **Auto-Formatting**:
  - Type "1." followed by space → Numbered list
  - Type "-" or "*" followed by space → Bulleted list
  - Type "[ ]" for checkbox item
- **Auto-Increment**:

  - Tab indents and changes to sub-level format

- **Auto-Terminate**:

#### **List Continuation Control**

- **Continue Previous**:

#### **List Styling**
- **Separator Customization**:
  - Between number and text: Period, parenthesis, dash, colon, nothing

- **Alignment**:
  - Number alignment: Left, right, centered

### **10.4 OpenType Typography Advanced**

#### **Variable Fonts**
- **Weight Axis**:
  - Continuous slider: 100 (Thin) to 900 (Black)
  - Smooth interpolation between weights
  - No limitation to predefined weights
- **Width Axis**:
  - Condensed to Extended
  - Percentage scale: 50% to 200%
- **Slant Axis** (Italic variation):
  - Continuous angle: 0° (upright) to 20° (italic)
- **Optical Size**:
  - Optimizes letterforms for display size
  - Auto-adjusts based on font size (if supported)
- **Custom Axes** (font-specific):
  - Grade, Expression, Casual, Cursive, etc.
  - Exposed as sliders for user control
- **Preset Instances**:
  - Named combinations of axis values
  - Example: "Display Bold", "Text Regular"

#### **Stylistic Sets**
- **Set Activation**:
  - Each set is a collection of alternate glyphs
  - Sets numbered 1-20 (ss01-ss20)
  - Multiple sets can be active simultaneously
- **Preview**:
  - Live preview showing before/after for each set
  - Affected characters highlighted
- **Character Variants (cv01-cv99)**:
  - Individual character alternates
  - Example: Different styles of lowercase 'a'
  - Choose per-character basis

#### **Ligature Control**
- **Standard Ligatures** (typically on by default):
  - fi, fl, ff, ffi, ffl
  - Improve readability and aesthetics
- **Discretionary Ligatures**:
  - ct, st, sp, etc.
  - Optional, more stylistic
- **Contextual Ligatures**:
  - Auto-applied based on surrounding characters
  - Font-specific logic
- **Historical Ligatures**:
  - Old-style ligatures (long s, etc.)
  - Rarely used, but available

#### **Number Formatting**
- **Lining Figures** (default):
  - All numbers same height as capitals
  - Good for all-caps text, tables
- **Old-Style Figures**:
  - Varied height (some below baseline)
  - More elegant in body text
- **Tabular Figures**:
  - Monospaced (all same width)
  - Perfect for tables, financial data
  - Columns align vertically
- **Proportional Figures**:
  - Variable width (natural spacing)
  - Better for running text
- **Slashed Zero**:
  - Zero with diagonal slash
  - Distinguishes from letter O

#### **Fractions**
- **Automatic Fractions**:
  - Typing "1/2" auto-converts to ½
  - Diagonal (½) or stacked (
  - Supports arbitrary fractions (e.g., 3/7)
- **Numerator/Denominator**:
  - Manual control over fraction components
  - Superscript numerator, subscript denominator

#### **Case-Sensitive Forms**
- **All Caps Adjustments**:
  - Punctuation positions adjust for all-caps
  - Example: Hyphen raises to center height
- **Small Caps**:
  - True small caps (designed, not scaled)
  - Option: All small caps or Small Caps From Caps
- **Petite Caps**: Smaller than small caps (if font supports)

## 11. INTEGRATION AND CROSS-ELEMENT CONSIDERATIONS

How text elements interact with other elements in the system.

### **11.1 Layering and Z-Order**

#### **Stacking Behavior**
- **Default**: Text elements added to canvas appear on top of existing elements
- **Z-Index**: Each element has numeric z-index (1 to 9999+)
- **Layer Panel**: Visual representation of stacking order
  - Top of list = top of canvas
  - Drag to reorder
  - Nested groups show hierarchy
- **Overlap Handling**:
  - Clicking selects topmost element
  - Alt+Click cycles through stack
  - Outline flash shows available elements

#### **Interaction with Other Element Types**
- **Text Above Images**: 
  - Text renders on top for legibility
  - Auto-shadow suggestion if contrast poor
- **Text Behind Shapes**:
  - Can send text to back for creative effects
  - Opacity adjustments maintain readability
- **Text Within Containers**:
  - Text can be child of frame/group
  - Inherits container transformations (move, rotate, scale)
  - Can be clipped by container bounds

### **11.2 Grouping Capabilities**

#### **Text-Only Groups**
- **Purpose**: Manage related text elements together
- **Behavior**:
  - Transform as unit (move, resize, rotate)
  - Individual elements remain editable
  - Common properties editable via right panel
- **Examples**: Headline + subtitle, bullet list + heading

#### **Mixed Element Groups**
- **Text + Images**: Common for layouts
- **Text + Shapes**: Badges, buttons, labels
- **Text + Charts**: Annotations, legends
- **Behavior**:
  - Text maintains editability within group
  - Text properties accessible even when group selected
  - Some properties show "Mixed" if incompatible types in group

#### **Nested Groups**
- **Hierarchy**: Groups within groups
- **Depth**: Up to 20 levels (though 3-4 typical)
- **Editing**:
  - Double-click group to enter
  - Breadcrumb navigation shows depth
  - Escape to exit group editing mode

### **11.3 Alignment and Distribution**

#### **Relative to Other Elements**
- **Align Edges**:
  - Left, right, top, bottom edges
  - Smart guides show alignment in real-time
- **Align Centers**:
  - Horizontal center (vertical line through center)
  - Vertical center (horizontal line through center)
- **Distribute Spacing**:
  - Horizontal: Even spacing left-to-right
  - Vertical: Even spacing top-to-bottom
  - Smart distribution: Based on element sizes

#### **Align to Canvas/Artboard**
- **Canvas Alignment**:
  - Center on canvas
  - Align to canvas edges
  - Margin-aware alignment (10px from edge)
- **Grid Snapping**:
  - Snap to grid lines (customizable grid: 10px, 20px, etc.)
  - Pixel-perfect alignment
  - Toggle snapping on/off (Cmd+')

#### **Baseline Alignment** (Text-Specific)
- **Align Baselines**: 
  - Multiple text elements align on text baseline (not element box)
  - Useful for different font sizes on same line
  - Example: "Sale" in large text + "50% off" in small text aligned on baseline

### **11.4 Responsive and Adaptive Behaviors**

#### **Canvas Resize Handling**
- **Fixed Position**: 
  - Text stays at absolute X, Y coordinates
  - May move off-canvas if canvas shrinks
- **Relative Position**:
  - Position as percentage of canvas size
  - Example: Always 10% from left, 20% from top
  - Maintains relative layout when canvas resizes
- **Anchored**:
  - Pinned to canvas edge (top, bottom, left, right, or corners)
  - Example: Footer text always 20px from bottom
- **Auto-Resize Text Box**:
  - Text box width/height adjusts with canvas
  - Maintains margin from edges
  - Text reflows within new dimensions

#### **Constraints**
- **Minimum/Maximum Size**:
  - Text box won't shrink below 50px width
  - Won't expand beyond 800px (prevents over-stretching)
- **Aspect Ratio Lock**:
  - Maintains proportions during resize
  - Useful for text in specific shapes

#### **Breakpoints** (for multi-device templates)
- **Device-Specific Text**:
  - Different font sizes for mobile vs. desktop
  - Show/hide text elements based on device
  - Example: Full paragraph on desktop, abbreviated on mobile
- **Automatic Adjustments**:
  - Font size scales down on smaller screens
  - Line height adjusts for readability
  - Margins compress on mobile

### **11.5 Export and Rendering**

#### **Text Rendering Modes**
1. **Editable Text** (for .docx, .pptx, editable PDFs):
   - Text remains selectable and editable
   - Font embedding required
   - File size smaller (text is just characters)
   
2. **Outlined/Vectorized** (for .svg, .eps, precise printing):
   - Text converted to vector paths
   - No longer editable as text
   - Font embedding not required
   - Preserves exact appearance
   - Larger file size
   
3. **Rasterized** (for .png, .jpg, .gif):
   - Text rendered as pixels
   - Resolution-dependent (set DPI: 72, 150, 300, etc.)
   - No fonts needed
   - File size depends on resolution and compression

#### **Font Embedding and Licensing**
- **Embed Fonts**:
  - Option to embed fonts in PDF/DOCX exports
  - Ensures appearance on systems without font installed
  - Licensing check: Warns if font license prohibits embedding
- **Subset Embedding**:
  - Only embed characters actually used (reduces file size)
  - Full embedding vs. subset toggle
- **Fallback Fonts**:
  - Specify fallback if primary font unavailable
  - System fonts recommended for web/email templates

#### **Export Quality Settings**
- **Text Anti-Aliasing**:
  - None (pixel-perfect, low quality)
  - Standard (smooth edges)
  - Subpixel (ClearType/LCD optimization)
  - Best quality (maximum smoothing, larger file)
- **DPI/Resolution** (for raster exports):
  - Screen (72 DPI): Web, digital use
  - Print (300 DPI): Professional printing
  - High-res (600+ DPI): Large format printing
- **Compression**:
  - Text usually benefits from lossless (PNG) or minimal compression
  - Avoid heavy JPEG compression on text (creates artifacts)

#### **Format-Specific Considerations**
- **PDF**:
  - Editable vs. flattened
  - PDF/A compliance for archival
  - Accessibility tags (for screen readers)
  - Form fields from text inputs
- **SVG**:
  - Text as `<text>` elements (scalable, selectable) or
  - Text as `<path>` elements (appearance guaranteed)
  - CSS styling embedded or inline
- **HTML/Email**:
  - Web fonts (Google Fonts, self-hosted)
  - Fallback font stack: "Arial, Helvetica, sans-serif"
  - Email-safe fonts (limited set for compatibility)
  - Responsive CSS for mobile
- **Image Formats**:
  - PNG: Best for text (lossless, transparency)
  - JPEG: Avoid for sharp text (compression artifacts)
  - GIF: Limited colors, avoid for multi-color text
  - WebP: Modern format, good quality and compression

### **11.6 Performance Implications**

#### **Large Text Elements**
- **Long Documents** (>10,000 words):
  - Pagination or virtual scrolling for editing
  - Lazy rendering: Only visible portion rendered
  - Background spell-check (doesn't block UI)
- **Many Text Elements** (>500 on canvas):
  - Warning if approaching limit
  - Performance mode: Simplifies rendering during pan/zoom
  - LOD (Level of Detail): Lower quality during interactions, high quality when static

#### **Complex Effects**
- **3D, Shadows, Gradients**:
  - GPU acceleration where possible
  - Option to disable effects preview during editing (for low-end hardware)
  - Render proxies: Low-res preview, full render on export
- **Path Text**:
  - Complex paths (100+ anchor points) may slow rendering
  - Simplify path suggestion
  - Pre-render for export

#### **Optimization Strategies**
- **Rasterize Complex Text**:
  - Convert heavily-styled text to image
  - Reduces rendering overhead
  - Trade-off: Loses editability
- **Reduce Shadow/Blur Quality**:
  - Lower quality during editing
  - High quality for final export
- **Limit Undo History**:
  - Keep last 50 actions (vs. unlimited)
  - Clears old history to free memory

## 12. BEST PRACTICES AND COMMON PATTERNS

Guidance on effective text element usage, drawn from Microsoft Word, Canva, and modern design principles.

### **12.1 Typography Best Practices**

#### **Font Selection**
- **Readability First**:
  - Body text: Sans-serif (Arial, Helvetica, Roboto) or serif (Georgia, Times) at 14-18px
  - Headings: More decorative allowed, but still legible
  - Avoid overly stylized fonts for body text
- **Font Pairing**:
  - Maximum 2-3 fonts per design
  - Pair serif headings with sans-serif body (or vice versa)
  - Similar x-heights for visual harmony
  - Contrast weights (light body, bold headings)
- **Hierarchy Through Size**:
  - H1: 2.5-3× body size
  - H2: 2× body size
  - H3: 1.5× body size
  - Consistent scale ratio (e.g., 1.5× per level)

#### **Line Length and Spacing**
- **Optimal Line Length**:
  - 50-75 characters per line for readability
  - Too short: Choppy reading
  - Too long: Eye strain, losing place
- **Line Height**:
  - Body text: 1.4-1.6× font size
  - Headings: 1.1-1.3× (tighter)
  - Lists: 1.6-1.8× (more space)
  - Adjust for font: Condensed fonts need more line height
- **Paragraph Spacing**:
  - 0.5-1× line height between paragraphs
  - Or use first-line indent (20-30px) instead of spacing
  - Don't use both indent and spacing

#### **Alignment and Justification**
- **Left Align** (most common):
  - Easiest to read for Western languages
  - Uneven right edge (ragged right) is fine
- **Center Align**:
  - Headings, titles, short text
  - Poor for long paragraphs (hard to find line start)
- **Right Align**:
  - Rare, mostly for design effect
  - Captions aligned to right edge of image
- **Justify**:
  - Formal documents, newspapers
  - Requires hyphenation to avoid large gaps
  - Can create "rivers" of white space (avoid)

#### **Color and Contrast**
- **High Contrast for Readability**:
  - Dark text on light background (or vice versa)
  - Minimum 4.5:1 contrast ratio (WCAG AA)
  - 7:1 for better accessibility (WCAG AAA)
- **Color Meaning**:
  - Red: Errors, warnings, stop
  - Green: Success, go, positive
  - Blue: Links, information, trust
  - Use consistently throughout design
- **Avoid**:
  - Low contrast (light gray on white)
  - Red/green only for critical info (colorblind users)
  - Busy backgrounds behind text

### **12.2 Layout and Composition**

#### **White Space (Negative Space)**
- **Importance**: 
  - Gives text room to breathe
  - Improves comprehension
  - Creates visual hierarchy
- **Application**:
  - Margins: Minimum 50px from canvas edge
  - Padding: 20-30px inside text boxes
  - Between sections: 40-60px vertical space
  - Around headlines: More space above than below (shows heading belongs to following content)

#### **Grid Systems**
- **Columns**:
  - Single column: 600px max width (readability)
  - Two columns: 300-400px each
  - Three+ columns: Magazine/newspaper style
  - Gutter: 20-40px between columns
- **Baseline Grid**:
  - All text aligns to invisible horizontal grid
  - Grid unit: Line height (e.g., 24px)
  - Creates rhythm and consistency
  - Especially important for multi-column layouts

#### **Visual Hierarchy**
- **Size**: Larger = more important
- **Weight**: Bold = emphasis
- **Color**: Bright/saturated = attention
- **Position**: Top-left = first seen (in Western layouts)
- **Spacing**: Isolated = important
- **Typographic Hierarchy Example**:
  1. Main headline: 48px, bold, dark
  2. Subheading: 24px, regular, medium gray
  3. Body: 16px, regular, dark gray
  4. Caption: 12px, italic, light gray

### **12.3 Common Design Patterns**

#### **Hero Section**
- **Structure**:
  - Large headline (48-72px)
  - Subheading or tagline (24-36px)
  - Call-to-action button with text
  - Background image with text overlay
- **Best Practice**:
  - High contrast for text over image
  - Consider gradient overlay on image for better readability
  - Limit to 1-2 sentences in headline

#### **Card Layouts**
- **Elements**:
  - Card title (H3, 24px)
  - Description text (16px, 2-3 lines)
  - Metadata (date, category - 14px, gray)
- **Spacing**:
  - 20px padding inside card
  - 16px between title and description
  - 8px between description and metadata

#### **Forms and Input Labels**
- **Label Positioning**:
  - Above input (most common, mobile-friendly)
  - Left of input (compact, desktop)
  - Inside input as placeholder (accessible only if label remains visible)
- **Hierarchy**:
  - Label: 14px, medium weight
  - Input text: 16px, regular
  - Helper text: 12px, gray
  - Error text: 12px, red

#### **Callouts and Quotes**
- **Block Quote**:
  - Larger text (120-150% of body)
  - Indented or bordered
  - Italic or contrasting font
  - Attribution below or to side
- **Pull Quote**:
  - Even larger (200-250%)
  - Centered or floated
  - Decorative quotation marks
  - Used to highlight key phrase from text

#### **Lists in Design**
- **Bulleted Lists**:
  - Feature lists
  - Non-sequential information
  - Keep items parallel (same grammatical structure)
  - Limit to 7±2 items for scannability
- **Numbered Lists**:
  - Steps, instructions, rankings
  - Sequential information
  - Use when order matters
- **Icon Lists**:
  - Replace bullet with icon
  - Icons should be relevant (checkmark = completed, star = featured)
  - Align icons for clean look

### **12.4 Workflow Recommendations**

#### **Efficient Text Creation**
1. **Start with Structure**:
   - Add all headings first
   - Fill in body text
   - Add lists, quotes, captions last
2. **Use Styles Early**:
   - Define text styles at project start
   - Apply styles to all text
   - Makes global changes easy later
3. **Placeholder Text**:
   - Use lorem ipsum for layout
   - Replace with real content when available
   - Check layout doesn't break with real text (length variations)

#### **Editing Workflow**
1. **Content Pass**: Write/edit text content, ignore formatting
2. **Formatting Pass**: Apply styles, adjust spacing, alignment
3. **Polish Pass**: Fine-tune kerning, widows/orphans, hyphenation
4. **Review**: Check spelling, grammar, consistency
5. **Accessibility Check**: Contrast, alt text, reading order

#### **Collaboration**
- **Version Control**:
  - Name versions: "ProjectName_v1_2025-02-06"
  - Save major milestones
  - Use "Save As" before major changes
- **Comments/Annotations**:
  - Use comment feature for feedback
  - Tag collaborators for specific issues
  - Mark comments resolved when addressed
- **Style Libraries**:
  - Share style libraries across team
  - Brand guidelines in styles
  - Consistent application across projects

### **12.5 Accessibility Best Practices**

#### **Text-Specific Accessibility**
- **Alt Text**:
  - Every important text element should have alt text
  - Describe content and purpose
  - Keep under 250 characters
  - Decorative text can have empty alt text
- **Reading Order**:
  - Ensure logical tab order/reading sequence
  - Headings → Subheadings → Body → Lists
  - Test with screen reader
- **Language Tags**:
  - Set language attribute for correct pronunciation
  - Especially important for multilingual documents
- **Semantic Structure**:
  - Use heading levels correctly (H1 → H2 → H3, not H1 → H3)
  - Proper list markup for lists
  - Don't use bold as heading substitute

#### **Color and Contrast**
- **Check All Text**:
  - Use contrast checker on all text/background combinations
  - Include text in images
  - Consider different color blindness types
- **Don't Rely on Color Alone**:
  - Links: Underline + color
  - Errors: Icon + color + text
  - Status: Label + color

#### **Font and Sizing**
- **Minimum Sizes**:
  - Body text: 16px minimum
  - Mobile: 14px absolute minimum
  - Avoid text smaller than 12px
- **Resizable**:
  - Support browser zoom (200%+)
  - Layout shouldn't break when text enlarged
  - Test at 200% zoom

### **12.6 Performance Optimization**

#### **Reduce Element Count**
- **Combine Text**:
  - Multiple small text elements → One text box with line breaks
  - Reduces overhead of managing many elements
- **Limit Effects**:
  - Not every text needs shadow/3D
  - Effects on key elements only
  - Disable effects during editing if slow

#### **Font Loading**
- **Preload Critical Fonts**:
  - Load fonts used above-the-fold first
  - Lazy load decorative fonts
- **Font Subset**:
  - Only include characters needed
  - Latin-only if no special characters
  - Reduces font file size 50-90%
- **Fallbacks**:
  - Always specify fallback fonts
  - Similar metrics to reduce layout shift

#### **Text Rendering**
- **Avoid Over-Nesting**:
  - Deep group hierarchies slow rendering
  - Flatten groups when possible
- **Cache**:
  - Rasterize static text that never changes
  - Cache rendered output for reuse
  - Clear cache if text edited

### **12.7 Common Mistakes to Avoid**

#### **Typography Mistakes**
- **Too Many Fonts**: Stick to 2-3 maximum
- **All Caps Overuse**: Hard to read, use sparingly
- **Poor Contrast**: Light gray on white is not accessible
- **Inconsistent Spacing**: Different line heights randomly
- **Orphans and Widows**: Single word on last line, or single line at top of column (use widow/orphan control)
- **Justified Text Without Hyphenation**: Creates awkward gaps
- **Center-Aligned Body Text**: Hard to read, use left-align
- **Fake Bold/Italic**: Use actual bold/italic fonts, not browser rendering

#### **Layout Mistakes**
- **No White Space**: Cramming too much text
- **Unequal Margins**: Asymmetric margins that look accidental
- **Ignoring Alignment**: Elements not aligning to grid or each other
- **Inconsistent Hierarchy**: H2 larger than H1, etc.
- **Text Too Close to Edges**: Needs breathing room

#### **Accessibility Mistakes**
- **Missing Alt Text**: Critical for screen readers
- **Color as Only Indicator**: Color blindness issues
- **Low Contrast**: Below WCAG standards
- **Tiny Text**: Below 12px
- **No Keyboard Access**: Can't navigate without mouse

#### **Workflow Mistakes**
- **Not Using Styles**: Manual formatting everywhere (hard to change)
- **Editing Before Content Final**: Wasting time formatting placeholder text
- **No Backups**: Losing work due to no version control
- **Ignoring Feedback**: Not incorporating review comments
- **Skipping Proofreading**: Spelling/grammar errors in final

This comprehensive documentation covers every aspect of text elements in a graphics template design web application. From the initial selection in the left panel to the final export settings, each property, interaction, shortcut, and interface location has been thoroughly documented.
